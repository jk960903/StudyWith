TCP 와 UDP 의 차이점은 무엇인가요

TCP 신뢰성 보장 1:1 통신 방식 UDP 보다 느리다 
UDP 신뢰성이 보장되지 않음 데이터 그램 방식 빠르다 

그럼 TCP가 신뢰성이 보장된다 하는데 어떻게 되는가?

기본적으로 3way handshaking 과정을 통해 보장이 된다.


HTTP Method를 CRUD  아는대로 다 말해봐라
GET POST PUT DELETE 가기본적으로 자원에 대한 CRUD 를 담당하고 있습니다. 
GET 은 READ를 목적으로 사용하며 보통 자원에 대한 요구가 헤더(url)에 포함되는 특징이있습니다.
POST CREATE를 목적으로 사용하며 보통 자원에 대한 요구가 header에 포함되지 않고 Body에 포함되어 GET에 비해 보안성은 조금 더 높은편
PUT은 UPDATE를 담당하는 메소드입니다. 자원에 대한 요구가 바디에 포함됩니다.
보통 UPDATE를 사용하는데 PUT을 사용하지만 PATCH도사용합니다. 
PATCH와 PUT 메소드의 차이점은 자원의 대한 처리에 있습니다. 
PUT 메소드는 해당 자원의 위치에 있는 요구를 파라미터로 의해 전부다 바뀌게 됩니다.
예를 들어 
이름
나이
주소 가있을때 PUT메소드에 대한 파라미터로 이름만 들어온다면 이름만 업데이트 된후 나이 , 주소에 대한 데이터는 사라지게 됩니다.

반면에 PATCH 메소드는 들어있는 파라미터만 수정하게 됩니다. 
이름 : 변경
이름 , 나이 그대로

DELETE 메소드는 말그대로 자원을 삭제하는 메소드 입니다. 

여기에 TRACE OPTION HEAD 메소드가 있는데
OPTION은 서버에서 제공하는 메소드 들이 무엇인지 확인하는 메소드이고
HEAD는 헤더를 반환해주는 메서드 입니다. 
TRACE는 좀더 알아보기


웹브라우저에 주소를 입력해서 우리의 화면이 보이기 까지의 과정을 설명해보시오
처음에 사용자가 주소를 브라우저에 입력
그후 DNS 주소(uri)인지 아닌지 확인 (내부 자료인지 외부인지) 확인 후 DNS 주소를 IP주소로변환
해당 자원에 대해 서버에 HTTP 요청
서버에서는 확인후 요청이 정당하다면 그요청을 처리후 반환

자료구조 아는대로 다 말해보아라
스택 큐 힙 리스트 맵 셋 트리
스택 FILO
큐 FIFO
힙 완전 이진트리로 보통 우선순위 큐를 할때 구현을 합니다 맥스힙 최소힙
리스트 공통된 구조로 선형으로 이루어진 자료구조
맵 key value로 이루어진 자료구조로 탐색에 주로 많이 쓰임
셋 중복을 허용하지 않는 자료구조
트리 노드와 간선으로 이루어져있으며 순환을 허용하지 않는 그래프 구조
